1. What is amqp?

AMQP (Advanced Message Queuing Protocol) is an open standard networking protocol designed for passing business messages between different applications or organizations. In your event driven system, it acts as the universal language that allows your publisher and subscriber to communicate through a central message broker (in this case, RabbitMQ). By using this protocol, the programs remain decoupled. The publisher can send data (events) to the broker without needing to know who the subscriber is or if the subscriber is even currently running. This ensures that your system can handle traffic reliably, as AMQP manages the logistics of routing and storing messages in a queue until they can be processed.

2. What does it mean? guest:guest@localhost:5672 , what is the first guest, and what is the second guest, and what is localhost:5672 is for?

The connection string guest:guest@localhost:5672 is essentially the home address and key that your subscriber uses to find and log into the RabbitMQ broker. The first guest in the string represents the username, while the second guest represents the password. These are the default administrative credentials provided by RabbitMQ for development environments. Using these default settings simplifies the setup process while you are building the initial code, though in a production environment, you would eventually replace these with more secure, unique credentials.
<br>The latter portion of the string (localhost:5672) defines the physical location of the broker on your network. Localhost specifies that the RabbitMQ instance is running on the same computer as your code, inside the Docker container you started earlier. The number 5672 is the specific TCP port that RabbitMQ listens on for incoming AMQP traffic. Together, these components ensure that your Rust program knows exactly which door to knock on and what password to provide to begin listening for new events.
